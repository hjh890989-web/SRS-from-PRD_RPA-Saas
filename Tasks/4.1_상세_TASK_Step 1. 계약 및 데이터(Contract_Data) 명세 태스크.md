# Step 1. 계약 및 데이터(Contract/Data) 명세 태스크

이 문서는 FactoryAI MVP 개발의 첫 번째 단계인 **데이터베이스 스키마 설계, API 계약 정의, 그리고 테스트용 Mock 데이터 구축**에 관한 통합 명세서입니다.

## 구성 안내
- **1-A. 데이터베이스 스키마 & 마이그레이션 (DB-001~017)**: 애플리케이션의 핵심 데이터 구조 정의
- **1-B. API 계약 정의 (API-001~019)**: 프론트엔드와 백엔드 간의 인터페이스 규격(DTO/Zod) 정의
- **1-C. Mock 데이터 및 Mock API (MOCK-001~010)**: 초기 개발 가속화를 위한 시드 데이터 및 스텁 API 구축

- ADR: ADR-10 (Prisma ORM 중립 DB)
- 제약사항: CON-12 (Supabase Free 500MB DB + 1GB Storage)

### :white_check_mark: Task Breakdown (실행 계획)
- [ ] **1.** `npx prisma init` 실행 — `prisma/schema.prisma` 기본 파일 생성

- [ ] **3.** `.env.local` (dev) / `.env.cloud` (MVP) 환경변수 파일 구조 생성

- [ ] **4.** Prisma Client 생성 스크립트: `package.json`에 `"db:generate": "prisma generate"`, `"db:push": "prisma db push"`, `"db:migrate:dev": "prisma migrate dev"` 추가
