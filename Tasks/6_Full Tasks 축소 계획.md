## 2. 🚫 초보자 실패 확률이 높아 "제외"된 주요 대상

1. **E3 ERP 브릿지 (Mock 직접 연동)**: 다른 시스템의 스키마 변경을 추적하고 실시간으로 동기화하는 로직은 초보자에게 트랜잭션 및 타임아웃 헬(Hell)을 유발합니다. ➡️ **엑셀(CSV) 업로드로 대체 (정적 데이터 업로드)**
2. **E2-B XAI 이상탐지 & HITL 자동화 플로우**: AI가 품질 이상을 판단하고, 30분 미응답 시 윗선으로 자동 에스컬레이션되는 다중 에이전트/자동화 로직은 예외 처리와 디버깅이 매우 어렵습니다. ➡️ **과감히 삭제 (관리자가 화면을 보고 수동 판단하는 구조로 변경)**
3. **E6 보안 패키지 & E7 대시보드 스케줄러 발행**: CISO용 네트워크 모니터링, 월말에 데이터를 취합해 자동 메일을 보내는 기능은 복잡한 크론탭/배치 인프라가 필요합니다. ➡️ **접속 시 실시간 데이터를 보여주는 단순 대시보드(화면)로 축소**
4. **SVC (오프라인 서비스 지원) 14건**: 온보딩 상태 관리, 바우처 심사 플로우 등은 오프라인 비즈니스 로직을 소프트웨어로 과도하게 옮긴 것입니다. ➡️ **MVP에서는 Notion이나 별도 엑셀로 수동 관리 (전면 제외)**

## 3. 💡 PoC MVP 생존 핵심 PMF 시나리오 (Core Actions)

MVP의 본질인 "방문 → 핵심 행동 1개 → 전환/저장" 시나리오는 다음과 같이 단선화됩니다.

- **[PMF 1. 생산직 - 행동]** 로그인 → `음성/사진 캡처` (유일한 외부 연동 API 호출) → `텍스트/데이터로 변환되어 DB 저장` (끝)
- **[PMF 2. 관리자 - 행동]** 로그인 → ERP 엑셀 업로드 → 데이터 목록 조회/필터링 및 `간단한 수정(CRUD)` → `PDF 다운로드` (끝)
- **[PMF 3. 잠재고객 - 전환]** 랜딩 페이지 방문 → `ROI 계산기 폼 입력` → 결과 확인 → `리드(Lead) 정보 남기기(DB 저장)`

## 4. 📝 재구성된 PoC MVP Task List (약 40여 개 핵심 정리)

### Step 1. 기반 세팅 및 공통 (Foundation - 단일 저장소, 단순 배포)
- **`INIT-001`** Next.js 15 (App Router) + Tailwind CSS 초기화 및 Vercel 단일 배포 연결
- **`DB-001`** Prisma + Supabase (PostgreSQL) 단순 환경 세팅
- **`DB-002`** User(Admin/User 2단계), Factory/Log 기본 엔터티 스키마 및 마이그레이션
- **`AUTH-001`** NextAuth.js 기반 최소 수준의 이메일/비밀번호 로그인 모듈

### Step 2. PMF 1: AI 무입력 로깅 (외부 API 1개 활용)
- **`E1-CMD-001`** [Command] 음성 녹음 → Vercel AI SDK(Gemini) 호출 → 텍스트 반환 및 DB 저장
- **`E1-CMD-002`** [Command] 현장 사진 업로드 → Gemini Vision API 호출 → 주요 항목 추출 및 DB 저장
- **`E1-UI-001`** [UI] 모바일 웹 기반 단순 레코더 / 사진 업로드 컴포넌트

### Step 3. PMF 2: 데이터 수동 적재 및 리포팅 (정적 데이터 일괄 업로드)
- **`E3-CMD-001`** [Command] 엑셀(CSV) 파일 파싱 후 DB 일괄 Insert (Bulk Upload)
- **`E1-QRY-001`** [Query] 저장된 로그 및 엑셀 데이터 단순 목록 조회 (DataTable - 검색/필터 지원)
- **`E1-CMD-003`** [Command] 수집된 데이터 관리자 수동 수정 / 승인 (단순 CRUD)
- **`E2-CMD-001`** [UI/Command] 필터링된 화면 데이터를 브라우저 단에서 1클릭 PDF 생성 (`window.print` 또는 클라이언트 라이브러리)

### Step 4. PMF 3: ROI 진단 및 리드 수집 (전환 의도)
- **`E4-CMD-001`** [Command] 기업 정보 폼 입력 시 단순 산식 기반 ROI 예상치 계산
- **`E4-UI-001`** [UI] 랜딩 페이지에 노출될 ROI 계산기 뷰
- **`LEAD-001`** [Command] 결과 확인 후 "도입 문의하기" 클릭 시 이름/연락처 DB에 저장 (Lead 획득)

### Step 5. 관리 및 단순 대시보드
- **`ADMIN-001`** [UI] 누적 로깅 횟수, 리드(Lead) 접수 현황을 보여주는 단순 정적 대시보드 화면 
- **`TEST-001`** [Test] 핵심 3대 플로우(녹음 저장, 엑셀 업로드, ROI 계산)에 대한 수동 E2E 테스트
