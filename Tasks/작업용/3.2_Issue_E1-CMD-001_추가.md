## :dart: Summary
- **기능명**: [E1-CMD-001] 버튼 녹음 → Gemini STT API 음성→텍스트 변환 + LOG_ENTRY 저장 (status=PENDING)
- **목적**: 작업자가 녹음 버튼을 눌러 음성을 녹음하면 Gemini STT API를 통해 공정 상태를 텍스트로 변환하고, 변환 결과를 LOG_ENTRY 테이블에 `status=PENDING`으로 저장한다. 이를 통해 현장 수기 입력을 제거하고 결측률 40%+ → ≤5%로 감소시키는 Zero-Touch 패시브 로깅의 핵심 기능이다.

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 정상적인 음성 녹음 → 텍스트 변환 + 로깅**
- **Given**: 작업자(OPERATOR 역할)가 유효한 오디오 파일(PCM 16kHz, 3초, 2MB)과 유효한 `factory_id`, `line_id`를 보유한다
- **When**: 녹음 버튼을 눌러 `POST /api/v1/log-entries`로 오디오를 전송한다
- **Then**: Gemini STT API가 호출되어 텍스트로 변환된다. LOG_ENTRY 테이블에 `source_type=STT`, `status=PENDING`, `parsed_data`에 변환 텍스트가 저장된다. 201 Created + `{ id, status: "PENDING", captured_at }` 반환. 인식 정확도 ≥85%, 전 과정 지연 ≤5초.

**Scenario 2: Gemini API Rate Limit 도달 시 큐잉 처리**
- **Given**: 현재 Gemini API 호출이 15 RPM 한도에 도달한 상태이다
- **When**: 추가 STT 요청이 인입된다
- **Then**: In-memory Queue에 대기열 추가되고, 사용자에게 `202 Accepted` + `{ status: "QUEUED", estimated_wait_seconds }` 반환. '처리 중' 진행률 표시기 활성화. 최대 대기 60초 이내 처리 완료.

**Scenario 3: 유효하지 않은 오디오 파일 업로드**
- **Given**: 크기가 15MB인 오디오 파일 또는 0.3초 미만의 매우 짧은 오디오가 주어진다
- **When**: `POST /api/v1/log-entries`로 전송한다
- **Then**: LOG_ENTRY 생성 없이 `400 Bad Request` + `{ error: "INVALID_AUDIO", message: "오디오 파일이 유효하지 않습니다 (최대 10MB, 최소 0.5초)" }` 반환. 시스템 크래시 0건.

**Scenario 4: 미인가 사용자 접근 시도**
- **Given**: VIEWER 역할의 사용자가 로그인되어 있다
- **When**: `POST /api/v1/log-entries`로 STT 로깅을 요청한다
- **Then**: `403 Forbidden` 반환. 감사 로그에 미인가 접근 시도 기록. CISO 알림 ≤10초 발송.

**Scenario 5: Gemini STT API 타임아웃**
- **Given**: Gemini API가 5초 이내에 응답하지 않는다
- **When**: STT 변환 요청이 타임아웃된다
- **Then**: 1회 자동 재시도 후에도 실패 시, `503 Service Unavailable` + `{ error: "STT_TIMEOUT", message: "음성 인식 서비스가 일시적으로 응답하지 않습니다" }` 반환. 감사 로그에 타임아웃 이벤트 기록. 오 데이터 기록 0건.

## :gear: Technical & Non-Functional Constraints
- **성능**: STT 전 과정(API 호출 + DB 저장) p95 ≤5,000ms (REQ-NF-001)
- **Rate Limit**: Gemini Free Tier 15 RPM 엄수. In-memory Queue + Exponential Backoff 적용 (ASM-10)
- **Throttle 대기**: 최대 60초, 진행률 UI 필수 (REQ-NF-008)
- **보안**: RBAC OPERATOR 이상만 접근. Prisma 감사 로그 전수 기록 (REQ-NF-023)
- **AI 추상화**: Vercel AI SDK 인터페이스로 구현하여 환경변수만으로 Ollama/vLLM 전환 가능 (ADR-9)
- **서버 실행 시간**: Vercel Serverless ≤10초 제한 준수 (CON-07)
- **데이터 무결성**: 인간 승인(HITL) 없이 외부 발행 0건 보장. status=PENDING으로만 저장 (ADR-3)
