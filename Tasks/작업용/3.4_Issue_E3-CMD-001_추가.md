## :dart: Summary
- **기능명**: [E3-CMD-001] Mock ERP 테이블 Read-Only 동기화 구현 (Write 시스템 레벨 차단, 합의 테이블만 읽기)
- **목적**: 기존 ERP(더존/영림원)를 **절대 손상시키지 않는 비파괴형 브릿지**를 구현한다. MVP에서는 Supabase 내 Mock ERP 테이블에서 합의된 테이블(재고, 발주, 실적)만 Read-Only로 동기화하며, **Write를 시스템 레벨에서 100% 차단**한다. CIO의 "기존 ERP 건드리면 절대 안 된다"는 핵심 요구사항을 기술적으로 보장하는 안전 커넥터이다.

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 정상적인 Read-Only 동기화**
- **Given**: Mock ERP 테이블(`mock_erp_inventory`)에 100건의 재고 데이터가 존재하며, ERP_CONNECTION의 `approved_tables`에 해당 테이블이 포함되어 있다
- **When**: `POST /api/v1/erp/sync`로 `{ connection_id, table_names: ["mock_erp_inventory"], period: "2026-04" }`를 요청한다
- **Then**: Mock ERP 테이블에서 100건의 재고 데이터를 Read-Only로 읽어온다. FactoryAI 내부 테이블에 동기화 데이터가 적재된다. `{ sync_status: "SUCCESS", records_count: 100, errors: [] }` 반환. **Mock ERP 테이블에 대한 DB 변경(Write) 0건**. 동기화 시간 ≤5분.

**Scenario 2: Write 시도 → 시스템 레벨 차단**
- **Given**: 임의의 코드가 Mock ERP 테이블에 INSERT/UPDATE/DELETE를 시도한다
- **When**: Prisma Middleware 또는 DB RLS에 의해 Write 요청이 감지된다
- **Then**: Write가 **즉시 차단**된다 (3중 차단: TypeScript 컴파일 / Prisma Middleware / Supabase RLS). 감사 로그에 차단 이벤트 기록. CISO 알림 ≤10초. **DB 변경 0건**.

**Scenario 3: 미합의 테이블 접근 시도**
- **Given**: `ERP_CONNECTION.approved_tables`에 `mock_erp_payroll`이 포함되어 있지 않다
- **When**: 동기화 시 `table_names: ["mock_erp_payroll"]`을 요청한다
- **Then**: 합의 테이블 가드가 접근을 **즉시 차단**한다. `403 Forbidden` + `{ error: "UNAPPROVED_TABLE", message: "해당 테이블은 CISO 승인 대상에 포함되어 있지 않습니다" }` 반환. 감사 로그 기록 + CIO 알림.

**Scenario 4: 스키마 변경 감지**
- **Given**: Mock ERP 테이블 `mock_erp_inventory`에 신규 컬럼 `inspection_date`가 추가되었다
- **When**: 동기화가 실행되며 스키마 검증이 수행된다
- **Then**: 스키마 변경이 감지된다. `sync_status=SCHEMA_CHANGED` 상태가 기록된다. (`E3-CMD-004`에서 동기화 중단 + CIO 알림 처리) **데이터 손상 0건**.

**Scenario 5: RBAC 권한 검증 — ADMIN/OPERATOR만 동기화 실행**
- **Given**: VIEWER 역할의 사용자가 로그인되어 있다
- **When**: `POST /api/v1/erp/sync`를 요청한다
- **Then**: `403 Forbidden` 반환. RBAC에 의해 차단. 감사 로그에 미인가 접근 시도 기록.

## :gear: Technical & Non-Functional Constraints

- **성능**: 동기화 ≤5분/배치 (REQ-NF-005 — MVP: Mock DB 직접 조회)
- **합의 테이블**: CISO 사전 승인 테이블만 접근 가능 (approved_tables 화이트리스트)
- **보안**: RBAC ADMIN/OPERATOR만 동기화 실행 가능. Prisma 감사 로그 전수 기록 (REQ-NF-023)
- **연결 정보**: `ERP_CONNECTION.connection_string` 암호화 저장 (§6.2.9)
- **MVP 범위**: 실제 ERP 연동은 Phase 2. MVP에서는 Supabase 내 Mock ERP 테이블만 대상
- **확장성**: `IERPConnector` 인터페이스 교체만으로 실제 더존/영림원 커넥터 전환 가능 (ADR-2)
- **스키마 안전**: 스키마 변경 시 자동 동기화 중단으로 데이터 손상 0건 보장 (REQ-FUNC-022)
