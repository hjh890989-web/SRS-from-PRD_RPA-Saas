# Step 2. 기능 구현(Implementation) 명세 태스크

이 문서는 FactoryAI MVP의 핵심 비즈니스 로직 구현을 위한 통합 명세서입니다. 인증/인가, AI 추상화, 알림 서비스 및 각 에픽(E1~E7)과 HITL 안전 프로토콜을 포함합니다.

## 구성 안내
1.  **2-A. 인증/인가 (AUTH-001~004)**: RBAC 기반 보안 및 전수 감사 로그 기반 구축
2.  **2-B. AI 추상화 & 2-C. 알림 (AI/NOTI)**: Vercel AI SDK 연동 및 시스템 알림 서비스
3.  **2-D. E1 패시브 로깅 (E1-CMD/QRY/UI)**: STT 및 Vision 기반 현장 데이터 수집
4.  **2-E. E2 감사 리포트 (E2-CMD/QRY/UI)**: Lot 병합 및 브라우저 사이드 PDF 생성
5.  **2-F. E2-B 품질 XAI (E2B-CMD/QRY/UI)**: 한국어 이상 징후 설명 및 판단 이력 추적
6.  **2-G. E3 ERP 브릿지 (E3-CMD/QRY/UI)**: 비파괴형 Read-Only 동기화 및 엑셀 우회 경로
7.  **2-H~K. 기타 에픽 및 HITL (E4, E6, E7, HITL)**: ROI 진단, 보안 패키지, 성과 대시보드, 공통 안전 가드

# 2-A. 인증/인가 공통 (AUTH-001 ~ AUTH-004)

> **보안 원칙**: Fail-Close (장애 시 기본 차단) + 전 접근 전수 감사 기록

### [AUTH-001] NextAuth.js v5 설정 + Credentials Provider
- **목적**: 이메일/패스워드 기반 인증 및 JWT 세션 전략 구축 (Supabase 최적화).
- **핵심**: `User.role` 및 `factory_id`를 JWT 토큰에 주입하여 모든 API에서 사용 가능하게 함.

### [AUTH-002] RBAC 미들웨어 — 5역할 접근제어 매트릭스
- **목적**: 19개 API 및 페이지 라우트에 대한 중앙 집중식 권한 검증.
- **방어**: Middleware(Edge) + Route Handler Guard(2차)의 2중 검증 체계.

### [AUTH-003] Prisma Middleware 전수 감사 로그 기록
- **목적**: 모든 DB 쓰기 작업(C/U/D)을 `AUDIT_LOG` 테이블에 자동 기록.
- **특징**: 민감 정보 마스킹 및 `AsyncLocalStorage`를 이용한 요청 컨텍스트 전파.

### [AUTH-004] 미인가 접근 감지 → CISO 알림 (≤10초)
- **목적**: 보안 위반 시 즉시 차단과 동시에 CISO에게 실시간 알림 발송.

---

# 2-B. AI 추상화 & 2-C. 알림 서비스

### [AI-001] Vercel AI SDK + Gemini 초기 설정
- **목적**: 모델 종속성 제거 (Gemini/OpenAI 전환 가능 구조).
- **핵심**: `lib/ai/provider.ts`를 통한 모델 추상화 팩토리 구현.

### [AI-002] In-memory Queue & Rate Limiter
- **목적**: Gemini Free Tier (15 RPM) 제한 극복을 위한 호출 큐잉 및 백오프.

### [AI-003] '처리 중' 진행률 표시기 UI
- **목적**: AI 처리 지연(최대 60초) 시 사용자가 상황을 인지하도록 돕는 시각적 가이드.

---

# 2-D. E1 패시브 로깅 (STT + Vision)

### [E1-CMD-001~002] 음성/이미지 로깅 처리
- **음성(STT)**: 현장 음성을 Gemini로 파싱하여 `LOG_ENTRY` 생성.
- **이미지(Vision)**: 아날로그 계기판 촬영 이미지를 분석하여 값 객체 추출.

### [E1-CMD-003] 롤백/수정 및 승인
- PENDING 로그의 상태를 APPROVED/REJECTED로 변경하며 감사 로그 자동 기록.

---

# 2-E. E2 원클릭 감사 리포트 (Lot Merge + PDF)

### [E2-CMD-001] Lot 시간순 병합 알고리즘
- 타임스탬프 기준으로 파편화된 로그를 하나의 Lot 타임라인으로 정합성 있게 병합.

### [E2-CMD-002] 브라우저 사이드 PDF 생성
- `@react-pdf/renderer`를 사용하여 클라이언트 단에서 규제 양식 리포트 출력.

---

# 2-F. E2-B 품질 XAI 이상탐지

### [E2B-CMD-001] Gemini 기반 XAI 한국어 설명 생성
- 이상 징후에 대해 "왜 이상인지" 현장 반장도 읽기 쉬운 한국어로 자동 설명 생성.

### [E2B-CMD-003] XAI 헬스체크 및 수동 전환
- AI 모듈 장애 시 ≤5분 내에 수동 판단 모드로 자동 전환하여 시스템 영속성 보장.

---

# 2-G. E3 ERP 비파괴형 브릿지

### [E3-CMD-001] Read-Only 동기화 (3중 Write 차단)
- **차단**: TypeScript(ReadOnly) + Prisma Middleware + Supabase RLS의 3중 방어.

### [E3-CMD-004] ERP 스키마 변경 감지
- 외부 ERP 구조 변경 시 동기화 즉시 중단 및 CIO 즉시 알림 발송.

---

# 2-H~K. ROI, 보안, 대시보드 및 HITL 공통

### [E4] ROI 진단/결재기
- 독립형 ROI 계산 엔진과 정부 바우처 적합성 진단 5항목 구현.

### [E6] 전역 보안 가드
- Supabase RLS 정책 설정 및 CISO 전용 보안 체크리스트 자동 생성.

### [E7] 성과 가시화 대시보드
- 월말 COO/CFO 등 4인 페르소나별 맞춤 대시보드 자동 발행 및 NPS 설문.

### [HITL] 공통 안전 프로토콜
- **원칙**: 미승인 건 외부 발행 절대 차단, 고위험 액션 시 승인 ID 필수 검증.
- **에스컬레이션**: 30분 미처리 PENDING 건 COO 자동 에스컬레이션.
