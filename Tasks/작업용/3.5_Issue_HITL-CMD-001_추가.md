## :dart: Summary
- **기능명**: [HITL-CMD-001] APPROVAL status=PENDING 외부 발행 자동 차단 + 관리자 알림 ≤10초
- **목적**: AI가 생성한 모든 결과물(감사 리포트, XAI 이상 판단 등)이 인간 승인 없이 외부로 발행되는 것을 **시스템 레벨에서 100% 차단**한다. HITL(Human-in-the-Loop) 4대 원칙의 핵심인 "AI 단독 실행 0건"을 기술적으로 보장하는 안전 게이트웨이이며, FactoryAI 전체 시스템의 신뢰성과 규제 컴플라이언스의 근간이 되는 기능이다.

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: PENDING 상태 외부 발행 시도 → 자동 차단**
- **Given**: AUDIT_REPORT ID=`rpt-001`에 대한 APPROVAL 레코드가 `status=PENDING`이다
- **When**: `POST /api/v1/audit-reports`로 해당 리포트의 최종 발행(PDF 외부 공유 등)을 시도한다
- **Then**: 발행이 **즉시 차단된다** (차단 ≤1초). `403 Forbidden` + `{ error: "PUBLICATION_BLOCKED", reason: "PENDING_APPROVAL" }` 반환. 관리자 알림이 **≤10초** 내 발송된다. 감사 로그에 차단 이벤트 기록.

**Scenario 2: APPROVAL 레코드 미존재 → 발행 차단**
- **Given**: AUDIT_REPORT ID=`rpt-002`에 대한 APPROVAL 레코드가 아예 존재하지 않는다
- **When**: 해당 리포트의 외부 발행을 시도한다
- **Then**: **즉시 차단**. `403 Forbidden` + `{ error: "PUBLICATION_BLOCKED", reason: "NO_APPROVAL_RECORD" }` 반환. "승인 프로세스를 먼저 시작해 주세요" 안내. 감사 로그 기록.

**Scenario 3: APPROVED 상태 → 정상 발행**
- **Given**: AUDIT_REPORT ID=`rpt-003`에 대한 APPROVAL이 `status=APPROVED`, `reviewer_id=user-001`(유효한 AUDITOR)이다
- **When**: 해당 리포트의 외부 발행을 시도한다
- **Then**: 발행이 **정상 통과**된다. 200 OK 반환. 감사 로그에 `PUBLICATION_ALLOWED` 이벤트 기록 (승인자, 리소스, 시점 포함).

**Scenario 4: REJECTED 상태 → 발행 차단 + 재요청 안내**
- **Given**: AUDIT_REPORT ID=`rpt-004`에 대한 APPROVAL이 `status=REJECTED`이다
- **When**: 해당 리포트의 외부 발행을 시도한다
- **Then**: **즉시 차단**. `403 Forbidden` + `{ error: "PUBLICATION_BLOCKED", reason: "REJECTED", message: "승인이 거부되었습니다. 수정 후 재승인을 요청하세요" }` 반환.

**Scenario 5: 발행 게이트 우회 시도 (API 직접 호출)**
- **Given**: 개발자/악의적 사용자가 발행 게이트를 우회하여 직접 PDF 다운로드 API를 호출한다
- **When**: 미승인 리소스에 대한 직접 접근이 감지된다
- **Then**: 우회 시도가 차단된다. `{ event_type: "PUBLICATION_BYPASS_ATTEMPT" }` 감사 로그 기록. **CISO 알림 즉시 발송**. AI 단독 실행 0건 보장.

**Scenario 6: 관리자 알림 SLA 준수**
- **Given**: Scenario 1에서 발행 차단이 발생한다
- **When**: 알림 발송 프로세스가 트리거된다
- **Then**: ADMIN 역할 사용자 + 지정 reviewer에게 알림이 **≤10초** 내 전달된다. 알림에 차단 유형, 리소스 ID, 승인 필요 안내가 포함된다.

## :gear: Technical & Non-Functional Constraints
- **차단 SLA**: 발행 시도 감지 → 차단 ≤1초 (REQ-FUNC-040)
- **알림 SLA**: 차단 감지 → 관리자 알림 발송 ≤10초 (REQ-FUNC-040)
- **AI 단독 실행**: 인간 승인 없이 외부 발행되는 건 **절대 0건** (REQ-FUNC-015, ADR-3)
- **보안**: 발행 게이트 우회 시도 감지 + CISO 즉시 알림 (REQ-FUNC-034)
- **감사 추적**: 차단/통과/우회 시도 전수 감사 로그 기록 (REQ-NF-023)
- **RBAC 연동**: 승인 권한은 역할별 제한 (ADMIN, AUDITOR만 최종 승인 가능)
- **무중단**: 게이트 미들웨어 장애 시 **fail-close** (차단 기본) — 장애 시에도 미승인 발행 0건

> [!CAUTION]
> **Fail-Close 원칙**: 발행 게이트 미들웨어에 오류 발생 시(DB 조회 실패, 타임아웃 등) 기본 동작은 **차단(BLOCK)**이다. 
> 절대로 fail-open(통과)으로 구현하지 않는다. 이는 HITL 원칙의 핵심이며, AI 단독 발행 0건을 보장하는 최후의 안전장치이다.
