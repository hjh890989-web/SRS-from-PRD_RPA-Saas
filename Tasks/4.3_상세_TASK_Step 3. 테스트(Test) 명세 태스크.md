# Step 3. 테스트(Test) 명세 태스크

이 문서는 FactoryAI MVP의 품질 보증을 위한 통합 테스트 명세서입니다. 모든 태스크는 완료 조건(AC)을 기반으로 하며, Jest(단위/통합)와 Playwright(E2E)를 사용합니다.

## 테스트 전략 및 원칙
- **전략**: 모든 AC/NAC를 자동화 테스트 코드로 구현하여 회귀 테스트 보장.
- **도구**: Jest, Playwright, ts-jest.
- **위치**: 단위/통합 테스트는 `src/__tests__/`, E2E 테스트는 `e2e/`.

---

# 3-A. E1 패시브 로깅 테스트 (TEST-E1)

### [TEST-E1-001] STT 음성→텍스트 변환 정확도/지연
- **목적**: Gemini STT 파이프라인의 정확도(≥85%) 및 응답 시간(p95 ≤ 5s) 검증.
- **시나리오**: 명확한 음성, 소음 환경, 응답 지연 시뮬레이션.

### [TEST-E1-002] Vision 이미지 파싱 정확도/지연
- **목적**: Gemini Vision을 통한 계기판 값 추출 성공률(≥85%) 및 응답 시간(p95 ≤ 5s) 검증.

### [TEST-E1-003] 승인/거부 반영 및 감사 로그
- **목적**: 상태 변경 반영 속도(≤1s) 및 감사 로그(`AUDIT_LOG`) 자동 기록 여부 검증.

---

# 3-B. E2 감사 리포트 테스트 (TEST-E2)

### [TEST-E2-001] Lot 병합 및 PDF 생성
- **목적**: 시간순 병합 정확도(≥99%) 및 `@react-pdf/renderer`를 통한 PDF 생성 검증.

### [TEST-E2-002] 결측치 감지 및 보완 알림
- **목적**: 필수 항목 누락 시 감지율(≥95%) 및 알림 발송 지연(≤30s) 검증.

---

# 3-C. E2-B 품질 XAI 이상탐지 테스트 (TEST-E2B)

### [TEST-E2B-001] XAI 한국어 설명 생성 및 구조
- **목적**: 설명 생성 지연(≤3s), JSON 구조 준수, 100% 한국어 출력 검증.

### [TEST-E2B-002] AI 단독 실행 전면 차단 (HITL)
- **목적**: 인간 승인 없는 고위험 액션 시 시도 100% 차단 및 보안 이벤트 알림 검증.

### [TEST-E2B-005] 30분 미처리 에스컬레이션
- **목적**: PENDING 건 방치 시 COO 자동 에스컬레이션 플로우 검증.

---

# 3-D. E3 ERP 브릿지 테스트 (TEST-E3)

### [TEST-E3-001] Read-Only 3중 차단 검증
- **목적**: TS + Prisma Middleware + RLS를 통한 ERP 테이블 쓰기 시도 100% 차단 검증.

### [TEST-E3-004] 스키마 변경 감지 및 동기화 중단
- **목적**: 외부 ERP 구조 변경 시 즉시 중단 및 CIO 알림 발송 검증.

---

# 3-E~G. ROI, 보안, 대시보드 테스트

### [TEST-E4] ROI 계산 정확도
- **목적**: 벤치마크 대비 정확도(±10%) 및 계산 속도(≤3s) 검증.

### [TEST-E6] RBAC 접근 권한 매트릭스
- **목적**: 5개 역할 × 19개 API 접근 제어 일치성(100%) 검증.

### [TEST-E7] 대시보드 월말 자동 발행
- **목적**: 발행 트리거 시 4인 페르소나별 데이터 생성 및 렌더링 지연(≤5s) 검증.

---

# 3-H. HITL 안전 프로토콜 테스트 (TEST-HITL)

### [TEST-HITL-001] 미승인 리포트 발행 차단
- **목적**: `status=PENDING` 리포트 다운로드 시도 시 즉시 차단(≤1s) 검증.

### [TEST-HITL-003] AI 헬스체크 실패 시 수동 전환
- **목적**: 3회 실패 시 `MANUAL_FALLBACK` 전환 및 복구 지연(30분) 검증.
